# DeepAgents Integration

One of the fundamental points to understand the power of **OpenWiki** is realizing that it is not just a standalone tool, but is entirely built on top of LangChain's **DeepAgents** framework.

## What is DeepAgents?

**DeepAgents** is an open-source Agent Harness specifically designed to build AI agents capable of executing complex and long-running tasks. The framework is built on top of LangGraph and LangChain.

DeepAgents features the following capabilities:
1. **Execution Environment:** Support for tools, virtual filesystem, and code execution.
2. **Context Management:** Summarizing long conversations, offloading large results, and using memory.
3. **Delegation:** The ability to break down complex tasks and spawn sub-agents to solve them in parallel.
4. **Steering:** The ability to pause the agent to request human approval (Human-in-the-loop).

## The Relationship Between OpenWiki and DeepAgents

Since OpenWiki is built on DeepAgents, it inherits all of its powerful capabilities. When you run OpenWiki to create documentation (Code Brain) or a personal memory (Personal Brain), what actually happens is:

1. A dedicated "Deep Agent" is created for this task.
2. This agent uses the Virtual Filesystem tools of DeepAgents to read code or fetch data from Connectors.
3. If the project is massive, the agent might divide the task (Delegation) to document different parts of the project in parallel.
4. Thanks to DeepAgents' robust context management, OpenWiki can handle massive code repositories without exceeding the language model's Context Window.

## Integrating OpenWiki with DeepAgents in Your Projects

If you are building your own agent using the `deepagents` library (whether in Python or TypeScript), you can leverage the wiki created by OpenWiki as part of your agent's context.

### Practical Example (Python):

Suppose you used `openwiki code --init` to create an `openwiki/` folder in your project. Now you want to build your own agent using DeepAgents to review code based on this documentation.

```python
from deepagents import create_deep_agent

# 1. Read the documentation generated by OpenWiki
with open("openwiki/Architecture.md", "r") as f:
    architecture_context = f.read()

# 2. Setup the agent's system prompt including the context
system_prompt = f"""
You are an expert code reviewer.
Before reviewing any code, please consider the project's architecture guidelines:

<architecture_guidelines>
{architecture_context}
</architecture_guidelines>

Ensure all your suggestions align with these guidelines.
"""

# 3. Create the agent using DeepAgents
agent = create_deep_agent(
    model="anthropic:claude-sonnet-4-6",
    system_prompt=system_prompt,
    # You can add tools to read additional wiki files if needed
)

# 4. Run the agent to review specific code
result = agent.invoke({
    "messages": [
        {"role": "user", "content": "Please review the new authentication logic in auth.py"}
    ]
})

print(result)
```

### The Benefit of This Integration:
When you link OpenWiki's output to your DeepAgents-built agent, you give your agent a ready-made "map" of the project. Instead of your DeepAgents agent having to search through all files using system tools, it starts immediately possessing the correct architectural understanding.

Additionally, since OpenWiki is built on DeepAgents, it natively supports tracing via **LangSmith**. If you provide a `LANGCHAIN_API_KEY`, you can see every step the OpenWiki agent took while building the documentation.
