# Practical Notes and Tips

Here are some important notes and tips when working with OpenWiki to get the best results:

## 1. Documentation is for AI, Not Humans
Always remember that the documentation generated by OpenWiki is designed to be "digestible" by Large Language Models (LLMs). The documentation might seem brief or dry to a human developer compared to Confluence docs, for example, but it is perfect for an AI agent because it provides direct technical information without fluff.

## 2. Model Selection
- **For complex tasks and massive repositories:** It is preferable to use very powerful models like `gpt-5.5` or `claude-sonnet-4-6`.
- **For simple updates:** You can use faster and cheaper models like `gpt-5.4-mini`.
- You can change the model used at any time via environment variables or through the `~/.openwiki/.env` settings.

## 3. Handling Directive Files (AGENTS.md / CLAUDE.md)
OpenWiki automatically adds a directive line in `AGENTS.md` and `CLAUDE.md` files to tell the agent to read the documentation.
**Tip:** Do not put the entire documentation inside these files! The goal of OpenWiki is to keep the documentation in a separate `openwiki/` folder, and tell the agent *where* the documentation is so it can fetch it when needed, to avoid consuming the Context Window in every message.

## 4. Data and Connectors Security
When using the **Personal Brain** mode and linking your accounts (Gmail, Notion, X):
- Secret Tokens are saved locally only on your machine in the `~/.openwiki/.env` path.
- Never share this file and do not upload it to GitHub.
- The personal wiki is generated and saved locally in `~/.openwiki/wiki/`.

## 5. Using LangSmith for Tracing
Since OpenWiki is built on DeepAgents, it supports direct integration with LangSmith.
**Tip:** During the initial setup of the tool, agree to add a LangSmith key. This will allow you to see every step the agent takes while scanning the code, the files it read, and the decisions it made, which helps tremendously in Debugging.

## 6. Handling Provider Errors
If you encounter network or API provider errors (like Timeout or Rate Limit), OpenWiki has an automatic Retry system. The default is 3 attempts. You can increase it via:
```bash
OPENWIKI_PROVIDER_RETRY_ATTEMPTS=5 openwiki code --update
```
