# ملاحظات عملية ونصائح

إليك بعض الملاحظات والنصائح الهامة عند العمل مع OpenWiki للحصول على أفضل النتائج:

## 1. التوثيق موجه للذكاء الاصطناعي، وليس للبشر
تذكر دائماً أن التوثيق الذي تولده OpenWiki مصمم ليكون "سهل الهضم" للنماذج اللغوية (LLMs). قد يبدو التوثيق مختصراً أو جافاً بالنسبة للمطور البشري مقارنة بوثائق Confluence مثلاً، ولكنه مثالي للوكيل الذكي لأنه يعطيه المعلومات التقنية المباشرة بدون حشو.

## 2. اختيار النموذج المناسب (Model Selection)
- **للمهام المعقدة والمستودعات الضخمة:** يُفضل استخدام نماذج قوية جداً مثل `gpt-5.5` أو `claude-sonnet-4-6`.
- **للتحديثات البسيطة:** يمكنك استخدام نماذج أسرع وأرخص مثل `gpt-5.4-mini`.
- يمكنك تغيير النموذج المستخدم في أي وقت عبر المتغيرات البيئية أو عبر إعدادات `~/.openwiki/.env`.

## 3. التعامل مع ملفات التوجيه (AGENTS.md / CLAUDE.md)
تقوم OpenWiki تلقائياً بإضافة سطر توجيهي في ملفات `AGENTS.md` و `CLAUDE.md` ليخبر الوكيل بقراءة التوثيق. 
**نصيحة:** لا تضع التوثيق كاملاً داخل هذه الملفات! الهدف من OpenWiki هو إبقاء التوثيق في مجلد `openwiki/` منفصل، وإخبار الوكيل بـ *مكان* التوثيق ليقوم بجلبه عند الحاجة، وذلك لتجنب استهلاك نافذة السياق (Context Window) في كل رسالة.

## 4. أمان البيانات والموصلات (Connectors Security)
عند استخدام نمط **Personal Brain** وربط حساباتك (Gmail, Notion, X):
- يتم حفظ الرموز السرية (Tokens) محلياً فقط في جهازك في مسار `~/.openwiki/.env`.
- لا تشارك هذا الملف أبداً ولا تقم برفعه إلى GitHub.
- الويكي الشخصي يتم توليده وحفظه محلياً في `~/.openwiki/wiki/`.

## 5. استخدام LangSmith للتتبع (Tracing)
بما أن OpenWiki مبنية على DeepAgents، فإنها تدعم التكامل المباشر مع LangSmith. 
**نصيحة:** أثناء إعداد الأداة لأول مرة، وافق على إضافة مفتاح LangSmith. هذا سيسمح لك برؤية كل خطوة يقوم بها الوكيل أثناء مسح الكود، والملفات التي قرأها، والقرارات التي اتخذها، مما يساعد جداً في تصحيح الأخطاء (Debugging).

## 6. التعامل مع أخطاء مزود الخدمة
إذا واجهت أخطاء متعلقة بالشبكة أو مزود الـ API (مثل Timeout أو Rate Limit)، فإن OpenWiki تمتلك نظام إعادة محاولة تلقائي (Retry). الافتراضي هو 3 محاولات. يمكنك زيادتها عبر:
```bash
OPENWIKI_PROVIDER_RETRY_ATTEMPTS=5 openwiki code --update
```
